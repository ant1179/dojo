## Étapes

1. Lire l’issue GitHub
2. Identifier fichiers impactés
3. Implémenter la fonctionnalité
4. Ajouter tests unitaires + intégration
5. Vérifier CI (lint + test + build)
6. Mettre à jour documentation
7. Ouvrir PR

## Règles
- Ne jamais modifier l’architecture sans justification dans une PR
- Toujours inclure les cas d’erreur
- Priorité aux tests
